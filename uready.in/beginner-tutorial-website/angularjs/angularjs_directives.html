    AngularJS Directives

    Directives are the most important components of any AngularJS application. Although AngularJS ships with wide range of directives, you will often need to create application specific directives. AngularJS also lets you define your own directives. A directive is something that introduces new syntax. Directives are markers on a DOM element which attach a special behavior to it.

    If you have written an Angular application before, then you have used directives, whether you realized it or not. You might have used simple directives like ng-model, ng-repeat, ng-show. All these directives attach special behavior to DOM elements.

    AngularJS directives are extended HTML attributes with the prefix ng-

    1.  The ng-repeat directive repeats the specific element.

    2.  The ng-model directive conditionally shows an element.

    3.  The ng-app directive initializes an AngularJS application.

    4.  The ng-init directive initializes application data.


     

    Example

    <script>
          (function(angular) {
          'use strict';
          angular.module('demo', [])
          .controller('Controller', ['$scope', function($scope) {
          $scope.guru = {
              name: 'Vivekananda',
                 Religion: 'Hindu'
             };
        }])
    </script>
    <body ng-app="demo">
        <div ng-controller="Controller">
            <div my-guru> </div>
        </div>
    Run the file »

    The ng-model Directive

    The ng-model directive binds the value of HTML controls (input, select, textarea) to application data. Binding the view into the model, which other directives such as input, textarea or select require. Keeping the state of the control (valid/invalid). Setting related css classes on the element (ng-valid, ng-invalid, ng-dirty, ng-pristine, ng-touched, ng-untouched, ng-empty, ng-not-empty) including animations.

    The ng-app Directive

    The ng-app directive starts an AngularJS Application. It defines the root element. It automatically initializes or bootstraps the application when web page containing AngularJS Application is loaded. It is also used to load various AngularJS modules in AngularJS Application. The ng-app directive wil automatically initialize) the application when a web page is loaded.

    The ng-init Directive

    The ng-init directive initializes an AngularJS Application data. It is used to put values to the variables to be used in the application. The ng-init directive allows you to evaluate an expression in the current scope.

    The ng-repeat Directive

    The ng-repeat directive repeats html elements for each item in a collection.

     

    Example

    <div ng-app="" ng-init="names=[
         {name:'Vetri',qualification:'MCA'},
         {name:'Boss',qualification:'BE'},
         {name:'Selva',qualification:'MSC'}]">
    <ul>
         <li ng-repeat="x in names">
             {{ x.name + ',' + x.qualification }}
         </li>
    </ul>
    </div>

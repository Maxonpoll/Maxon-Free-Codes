    AngularJS Modules

    The angular.module is a global place for creating, registering and retrieving Angular modules.A module is a collection of configuration and run blocks which get applied to the application during the bootstrap process. A module is a collection of services, directives, controllers, filters, and configuration information. Modules are used to separate logics say services, controllers, application etc. and keep the code clean.

    Creating new modules in AngularJS

    The angular.module created by collection of AngularJS function.

     
    // create a new module
          var myModule = angular.module('myModule', []);
    // register a new service
          myModule.value('appName', 'MyCoolApp');

    Add the Directive

    At a high level, directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler ($compile) to attach a specified behavior to that DOM element (e.g. via event listeners), or even to transform the DOM element and its children.

     

    Example

    <div ng-app="myApp" vetbossel-directive > </div>
    <script>
    var app = angular.module("myApp", []);
    app.directive("vetbosseldirective", function() {
             return {
         template: "Don't give up"
             };
    });
    </script>

    Add the controller

    The Controllers used to develop the application in ng-controller directive.

     

    Example

    <div ng-app="myApp" ng-controller="vetbossel" >
    {{ name + " " + age }}
    </div>
    <script>
    var app = angular.module("myApp", []);
    app.controller("vetbossel", function($scope) {
            $scope.name = "vetri";
            $scope.age = "21";
    });
    </script>
